"사이트에 이미지를 넣는 방법은 두 가지가 있습니다. 태그를 사용하는 것과 태그에 background-image 속성을 추가하는 것. 두 가지 방법의 차이점과 각각 어떠한 경우에 사용하면 좋은지 설명해보세요."

위의 내용을 설명하기 위해서는 Semantic Web과 Semantic Tag의 이해가 필요한데요.

Semantic Web

  • Semantic(의미론적인) Web(HTML) 은 단순한 코드 형대가 아닌 코드 자체에도 의미가 있어서 사람과 기계가 함께 이해 할 수 있는 웹페이지입니다.

Semantic Tag

  • 과거에 non-semantic tag e.g., div, span등이 아닌 의미를 명확히 설명하는 semantic tag e.g., <form>, <table>, <timg>등이 있다.
non-semantic tag와 semantic tag비교
태그는 semantic tag고 background-image는 non-semantic tag인건 알겠는데 어떤 차이가 있는가?

background-image는 non-semantic tag라서 사람이라면 <div class="bg-img">를 보고 img인줄 알수 있겠지만 기계라면 이게 <div> 인지 image인지 알수가 없다 하지만 semantic tag인 <img>를 쓰면 사람도 기계도 이것이 image 인지 알수가 있어서 만약 사람들이 인터넷에 image를 검색을 할때 background-image보단 &itimg>태그를 쓴 image가 먼저 검색이 된다.

또한 위에서 고양이 사진에 마우스를 올려보면 첫번째 background-image는 "이미지를 다른 이름으로 저장..." 이 뜨지 않지만 semantic tag를 사용한 <img>를 쓰면 "이미지를 다른 이름으로 저장..."이 뜬다

background-image = 이미지로 인식 못함

<img> 태그 = 이미지로 인식, 저장, 복사, 검색 가능

<img> 태그를 사용하는게 좋은 경우

  1. 프린트가 필요한 경우
    • 백그라운드 이미지는 출력시 포함되지 않습니다
  2. 이미지에 문구가 사용되어 의미를 가지는 경우
    • 이미지에 의미가 있는 경우, alt 태그를 사용해 의미 설명 가능
  3. 이미지가 중요한 경우

background-image를 사용하는게 좋은 경우

  1. 일정부분만 선택해 보여줄 경우(예 - hover버튼)
    • 간단하게 css를 사용하여 이미지를 손 쉽게 변경 가능
  2. 이미지 위에 텍스트가 들어가는 경우
    • 이미지 위에 텍스트가 들어가고, 백그라운드로 중요도가 높지 않을 때
  3. 페이지 전체 출력시 이미지를 제거해야 되는 경우
    • 이미지가 출력되면 곤란한 경우거나 겹치는 경우
  4. CSS sprites를 사용해 이미지 속도를 향상시킬 경우
    • 다수의 img 태그를 통한 방법보다 CSS sprites 기법을 사용하면 이미지의 수정, 보관, 관리가 용이하고 미리 다른 아이콘 영역도 읽어들여 로딩 속도도 향상 가능
  5. 배경 전체에 확대해 보여줄 필요가 있는 경우
    • background-size 등의 속성을 사용해 배경 이미지를 간단하게 조작 가능

결론적으로

이미지가 필요할때 특별한 경우가 아니라면 <img> 태그를 사용하는게 좋다
