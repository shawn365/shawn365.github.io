위의 내용을 설명하기 위해서는 Semantic Web과 Semantic Tag의 이해가 필요한데요.

Semantic Web

  • Semantic(의미론적인) Web(HTML) 은 단순한 코드 형대가 아닌 코드 자체에도 의미가 있어서 사람과 기계가 함께 이해 할 수 있는 웹페이지입니다.

Semantic Tag

  • 과거에 non-semantic tag e.g., div, span등이 아닌 의미를 명확히 설명하는 semantic tag e.g., form, table, img등이 있다.
non-semantic tag와 semantic tag비교
태그는 semantic tag고 background-image는 non-semantic tag인건 알겠는데 어떤 차이가 있는가?
background-image는 non-semantic tag라서 기계가 이게 
 인지 image인지 알수가 없다 하지만 semantic tag인 를 쓰면 사람도 기계도 이것이 image 인지 알수가 있어서 만약 사람들이 인터넷에 image를 검색을 할때 background-image보단 태그를 쓴 image가 먼저 검색이 된다.
